## 1.1 Пользовательские сценарии

1. Узнать погоду
	**Описание:** Как пользователь я хочу узнать текущую погоду в выбранном мною городе.
	**Сценарий:**
	1. Пользователь открывает приложение.
	2. Вводит название города
	3. Приложение отправляет запрос к API погоды
	4. Приложение получает данные о погоде
	5. На основе погодных условий выбирается подходящий мем
	6. Пользователю отображается:
		- город
		- температура
		- мем
		- описание погоды
		- иконка (солнечно, облачно и т.д.)

2. Изменить город
	**Описание:** Как пользователь я хочу посмотреть погоду в другом городе.
	**Сценарий:**
		1. Пользователь вводит новый город
		2. Приложение запрашивает данные о погоде для нового города
		3. Отображается обновлённая информация о погоде и подходящий мем

3. Определение геолокации
	**Описание:** Как пользователь я хочу определять город по геолокации.
	**Сценарий:**
	1. Пользователь нажимает кнопку "Определить моё местоположение"
	2. Приложение запрашивает разрешение на доступ к геолокации
	3. Получает координаты
	4. Приложение отправляет координаты в API погоды
	5. Отображает погоду и мем

4. Поделиться погодой
	**Описание:** Как пользователь я хочу поделиться картинкой с погодой и мемом в мессенджерах или соцсетях.
	**Сценарий:**
	1. Пользователь нажимает кнопку "Поделиться"
	2. Приложение формирует карточку:
		- город
		- температура
		- описание
		- мем
	3. Приложение сохраняет карточку на устройство пользователя

5. Добавление новых мемов в базу 
	**Описание:** Как пользователь я хочу самостоятельно добавлять новые мемы в базу.
	**Сценарий:**
	1. Пользователь открывает форму для добавления мема
	2. Загружает изображение
	3. Пользователь указывает категорию погоды
		- жарко
		- холодно
		- дождь
		- снег
		- ветер
	4. Мем сохраняется в базе

6. Редактирование категории мемов в базе
	**Описание:** Как пользователь я хочу изменять категории у мемов.
	**Сценарий:**
	1. Пользователь открывает форму для изменения категории мема
	2. Пользователь указывает новую категорию погоды
		- жарко
		- холодно
		- дождь
		- снег
		- ветер
	3. Мем обновляется в базе

7. Удаление мемов из базы
	**Описание:** Как пользователь я хочу удалять мемы.
	**Сценарий:**
	1. Пользователь нажимает кнопку "Удалить мем"
	2. Мем удаляется из базы

## 1.3 Проектирование API

Backend предоставляет REST API ля взаимодействия с клиентской частью приложения. Данные передаются в формате JSON. Авторизация в API не используется.

1. **Получение данных о погоде и мема**

	**Endpoint**: GET /weather

	**Описание**: Возвращает текущие погодные условия для указанного города и соответствующий мем.

	**Параметры запроса**
	- Параметр: city
	- Тип: string
	- Описание: название города

	**Пример запроса**: GET /weather?city=London

	**Пример ответа:**
		```
			{
			  "city": "London",
			  "temperature": 18,
			  "feels_like": 16,
			  description":"cloudy",
			  "icon": "cloud",
			  "wind_speed": 7,
			  "meme": "rain_duck.jpg"
			}
			```
	 **Описание полей**
	- city — название города
	- temperature — температура в градусах Цельсия
	- description — краткое описание погодных условий (например, "облачно, сильный ветер")
	- icon — иконка для frontend (солнечно, облачно и т.д.)
	- wind_speed — скорость ветра
	- meme — имя файла или URL мема, соответствующего погоде
	
	**Возможные ошибки**
	-  400 — неправильный параметр запроса: `{ "error": "Invalid city parameter" }`
	- 404 — город не найден: `{ "error": "City not found" }`
	- 500 — внутренняя ошибка сервера: `{ "error": "Internal server error" }`

2. **Добавление нового мема
	
	**Endpoint**: POST /memes
	
	**Описание:** Позволяет пользователю загрузить новый мем в базу данных.
	
	**Параметры запроса**
	- image (file) — файл с изображением мема
	-  category (string) — категория погоды (дождь, снег и т.д.)
	
	**Пример ответа:**
		```
			{
			  "status": "success",
			  "message": "Meme added"
			}
		```
	**Описание полей**
	- image — файл изображения мема
	- category — категории погоды (дождь, снег и т.д.)
	
	**Возможные ошибки**
	-  400 — неправильный данные файла: `{ "error": "Invalid image file" }`
	- 500 — внутренняя ошибка сервера: `{ "error": "Internal server error" }`

3. **Оценка мема (опционально)**
	**Endpoint**: POST /memes/{id}/rate
	
	**Описание:** Позволяет пользователю поставить оценку мему.
	
	**Параметры запроса**
	- rating (number) — оценка мема (1-5, например)
	
	**Пример ответа:**
		```
			{
			  "status": "success",
			  "message": "Rating saved"
			}
		```
	**Возможные ошибки**
	-  400 — неправильный параметр rating: `{ "error": "Invalid rating" }`
	- 404 — мем не найден: `{ "error": "Meme not found" }`
	- 500 — внутренняя ошибка сервера: `{ "error": "Internal server error" }`

## 1.4 Модель данных

**База данных:** SQLite

**Сущности:**
1. WeatherMeme — мем, привязанный к погоде
		```
		{
		  "id": 1,
		  "category": "rain",
		  "file_name": "rain_duck.jpg",
		  "rating": 4.5
		}
		```
	- id — уникальный идентификатор мема (INTEGER, PRIMARY KEY)
	- weather_category — категория погоды (TEXT), например "rain", "snow", "hot"
	- file_name —  имя файла или URL изображения (TEXT)
	- (опционально) rating — средняя оценка мема (REAL) 

## 1.5 Ключевые технические решения

#### Frontend:

**Тип:** веб-приложение

**Технологии:** HTML/CSS/JavaScript

**Причины выбора:**
- Легко интегрируется с REST API FastAPI
- Кроссплатформенность: работает в любом браузере на ПК или мобильном устройстве
- Можно быстро реализовать интерфейс с отображением погоды, мемов и кнопкой "Поделиться"
- Позволяет добавить лёгкие анимации и интерактивные элементы без сложной сборки

#### Backend:

**Тип:** REST API

**Технологии:** Python (FastAPI)

**Причины выбора:**
- Простая реализация API для получения погоды и мемов
- Лёгкий запуск и отладка на локальной машине
- Поддержка JSON по умолчанию
- Можно быстро интегрировать внешние API, например OpenWeatherMap

#### База данных:

**Тип:** SQLite

**Причины выбора:**
- Лёгкая встроенная база, не требующая отдельного сервера
- Подходит для небольших приложений
- Простое резервное копирование (файл базы)
- Не требует сложной настройки сервера

#### Внешние API:

**OpenWeatherMap** для получения данных текущей погоды по городу или координатам
